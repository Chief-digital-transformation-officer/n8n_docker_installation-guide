# Установка n8n на чистый Linux-сервер

## Что нужно

- **VPS** — Ubuntu 22.04 / 24.04, минимум 2 CPU (два ядра) / 2 GB RAM (оперативной памяти)
- **Домен** — купленный у любого регистратора, например reg.ru, с возможностью редактировать DNS

---

## Шаг 1. Купить VPS и настроить домен

**VPS:** Beget, Selectel, Яндекс, Timeweb, Hetzner, Contabo — любой.

> **VPS и нейросети.** Если с сервера нужен доступ ко всем нейросетям без ограничений, чаще выбирают зарубежный VPS. Имейте в виду: если туда попадают персональные данные клиентов, формально это может противоречить законодательству Российской Федерации, если вы из России.
>
> У российских провайдеров возможности различаются. Например, **Beget** проксирует запросы к **OpenAI** — GPT обычно доступен; **Google Gemini** при этом часто недоступен «из коробки»: у него более жёсткие ограничения, и на Beget по умолчанию он обычно не проходит.

При покупке выбирайте Ubuntu 22.04 или 24.04. Сохраните себе IP сервера и ОЧЕНЬ ВАЖНО - пароль для доступа к сервере, он будет указан на этапе создания/покупки сервера или придет на почту. Это пароль от root пользователя linux сервера.


**Домен:** зарегистрировать у любого регистратора, например reg.ru для россии. В DNS-настройках домена создать **A-запись**:

```
Тип:  A
Имя:  @          (или @, или любой "поддомен", например у вас домен site.ru, если напишите строку в A запись pro - то привяжете n8n к поддомену pro.site.ru, к слову, таким образом можно переиспользовать ваш существующий домен и не нужно покупать отдельный. Просто берете домен вашего сайта, site.ru и добавляете поддомен, automations.site.ru , писать в название поддомена n8n не рекомендую, на это иногда разные сервисы будут ругаться)

Значение: <IP сервера>
TTL:  3600 (берем по умолчанию что там стоит, это просто время, как быстро обновится эта запись)
```

## Шаг 2. Подключиться к серверу

**macOS или Linux** — откройте приложение «Терминал».

**Windows (относительно новая система)** — в меню «Пуск» введите **WSL**, установите то, что предложит система; дистрибутив выберите первый из списка (**Debian** или **Ubuntu** — как покажет мастер). После установки запуск **WSL** из «Пуска» открывает тот же по смыслу терминал, что на Linux или macOS.

**Windows (чуть старше)** — [видео по настройке Windows и WSL](https://youtu.be/EH3yeiZ5JRo?t=1074) (примерно с 17-й минуты — установка и настройка WSL).

В терминале выполните команду

```bash
ssh root@<IP адрес вашего сервера>
```

вводим пароль от сервера


Вы окажетесь в домашней директории root (`/root`). Именно отсюда запускается скрипт.

---

## Шаг 3. Скачать и запустить скрипт установки

Скачайте скрипт прямо на сервер одной командой:

```bash
curl -fsSL https://raw.githubusercontent.com/Chief-digital-transformation-officer/n8n_docker_installation-guide/main/n8n_setup.sh -o n8n_setup.sh
```

Запустите скрипт:

```bash
bash n8n_setup.sh
```

Скрипт задаст два вопроса:

```
Домен (например, example.com): yourdomain.com
Email для SSL (Let's Encrypt):      вписываете любой свой  email
```

Больше ничего вводить не нужно. Установка занимает 2–5 минут.

---

## Что делает скрипт

| Шаг | Действие |
|-----|----------|
| 1 | Проверяет / устанавливает Docker и Docker Compose plugin |
| 2 | Генерирует случайные пароли и `N8N_ENCRYPTION_KEY` |
| 3 | Создаёт директории в `~/n8n/` для данных |
| 4 | Пишет `.env` с вашим доменом и сгенерированными секретами |
| 5 | Пишет `docker-compose.yml` (домен берётся из `.env`) |
| 6 | Пишет `healthcheck.js` и `init-data.sh` |
| 7 | Запускает стек: Traefik → PostgreSQL → Redis → n8n → n8n-worker |

### Что поднимается

```
Traefik      — реверс-прокси, порты 80/443, SSL от Let's Encrypt (автоматически)
PostgreSQL   — база данных n8n
Redis        — очередь задач
n8n          — основной инстанс
n8n-worker   — обработчик выполнений
```

### Структура данных на сервере

```
~/n8n/
├── .env                  ← все настройки и секреты
├── docker-compose.yml
├── healthcheck.js
├── init-data.sh
├── credentials.txt       ← логин и пароль n8n
├── traefik_data/         ← SSL-сертификаты
├── n8n_storage/          ← файлы и конфиг n8n
├── db_storage/           ← данные PostgreSQL
└── redis_storage/        ← данные Redis
```

## После установки

Открыть в браузере:

```
https://yourdomain.com
```

> SSL-сертификат выдаётся автоматически через Let's Encrypt. Если браузер показывает ошибку SSL сразу после установки — подождать 1–2 минуты и обновить страницу.

Создать учётную запись администратора (первый вход — регистрация).

---

## Управление стеком

```bash
cd ~/n8n

docker compose logs -f n8n          # логи n8n в реальном времени
docker compose logs -f n8n-worker   # логи worker
docker compose ps                   # статус контейнеров
docker compose restart              # рестарт всего стека
docker compose down                 # остановка
docker compose up -d                # запуск
```

### Обновление n8n

```bash
cd ~/n8n
docker compose pull
docker compose up -d
```

> Для воспроизводимых обновлений — зафиксируйте версию в `docker-compose.yml`:
> `image: docker.n8n.io/n8nio/n8n:1.XX.X`

---

## Резервная копия

Достаточно заархивировать две директории:

```bash
tar -czf n8n_backup_$(date +%F).tar.gz ~/n8n/n8n_storage ~/n8n/db_storage
```

`.env` хранить отдельно в надёжном месте — в нём `N8N_ENCRYPTION_KEY`, без которого бэкап не расшифровать.
